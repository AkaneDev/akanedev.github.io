RTM — Random Text Model Specification

A lightweight standard for constructing pseudo-LLMs that generate random or pattern-based responses without neural inference.

1. What is RTM?

Random Text Model (RTM) is a framework and API standard for creating lightweight, non-AI text generators that simulate conversational AI behavior using randomness, heuristics, or small prediction algorithms instead of large neural networks.

The goal of RTM is to provide a reproducible and fun way to design fake language models (like CatGPT) for experimentation, parody, or prototyping purposes.

2. Core Principles

  • No AI required: RTMs do not rely on deep learning, embeddings, or trained transformers.
  • Predictive illusion: They create the appearance of intelligence through randomness, keyword triggers, or templated logic.
  • Deterministic chaos: Outputs are pseudo-random but optionally seeded for reproducibility.
  • Compatibility: RTMs mimic the /v1/chat/completions format for easy integration with existing clients and UIs.

3. RTM API Specification

Endpoints

  • GET /v1/models — Lists available RTMs.
  • POST /v1/chat/completions — Generates a random or heuristic response to the given input.

Example Request

{
  "model": "rtm-base",
  "messages": [
    {"role": "user", "content": "hello there"}
  ]
}

Example Response

{
  "id": "chatcmpl-rtm",
  "object": "chat.completion",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "As a curious RTM, I calculated that 42% of 'hello there' is cat energy."
    },
    "finish_reason": "stop"
  }],
  "usage": {"prompt_tokens": 2, "completion_tokens": 14}
}

4. RTM Model Types

  • RTM-Base: Simple random templates and Markov chains.
  • RTM-Pattern: Regex or keyword-driven responses.
  • RTM-Weighted: Frequency-based word prediction.
  • RTM-Sim: Simulated reasoning using buzzwords or pseudo-logic.
  • RTM-Themed: Personality-based variants (e.g., CatGPT).

5. Behavior & Memory

RTMs may optionally maintain short-term context (“fake memory”) using simple data structures such as arrays, JSON objects, or random seed storage. This is not required but can improve conversational continuity.

Example behaviors include:

  • “Mood” randomization per session (e.g., happy, sleepy, chaotic).
  • Seeded pseudo-random responses for repeatable testing.
  • Context echoes — repeating parts of the prompt for illusion of understanding.

6. Example Implementation

def RTM_generate(prompt):
    import random
    moods = ["curious", "chaotic", "sleepy", "serious"]
    mood = random.choice(moods)
    templates = [
        f"As a {mood} RTM, I think '{prompt}' is fascinating.",
        f"{prompt}? Hmm... sounds philosophical.",
        f"I computed that 73% of '{prompt}' is fluff.",
        f"In another universe, '{prompt}' would make sense."
    ]
    return random.choice(templates)
